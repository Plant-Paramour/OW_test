## 1. 架构总览 & 数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                      Raw Kaggle obs (dict/attr)                 │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  world/observation.py: parse_observation()                      │
│  → GameState (统一格式)                                          │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                  ┌────────────┼────────────┐
                  ▼            ▼            ▼
┌─────────────────────┐ ┌──────────────────────┐ ┌──────────────────┐
│ world/fleet_tracker  │ │ world/combat.py      │ │ engine/          │
│ build_arrival_ledger │ │ simulate_planet_     │ │ prediction.py    │
│ → {pid: [(eta,own,s)]}│ │   timeline()         │ │ interception.py  │
└────────┬────────────┘ │ → timeline dicts      │ │ (aim_at 预计算)   │
         │              └────────┬─────────────┘ └────────┬─────────┘
         │                       │                        │
         └───────────┬───────────┼────────────────────────┘
                     │           │
                    ▼           ▼
┌─────────────────────────────────────────────────────────────────┐
│  features/builder.py: build_decision_matrix()                   │
│  → list[DecisionRow]  每行 = 一个 (源行星, 候选目标) 决策对       │
│     self(21) + cand(29) + global(16) + mask + action_info        │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  policy/model.py ✅ (Phase 5 完成)                                │
│  三个 Encoder → joint embedding → TargetHead / ShipHead / Value  │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  policy/action_head.py: sample_action() → (target_idx, ratio)   │
│  physics.py: estimate_arrival() → (angle, eta)                  │
│  → Kaggle action: [source_id, target_id, ships]                 │
└─────────────────────────────────────────────────────────────────┘
```

**核心设计原则：** Agent 从不输出角度。动作只有 `(source_planet, target_planet, ship_ratio)`，物理引擎自动计算 `atan2` 和 ETA。

### 2.4 interception.py — 拦截求解

```python
from src.engine.interception import aim_at, search_safe_intercept
```

| 函数 | 签名 | 返回值 | 说明 |
|------|------|--------|------|
| `aim_at` | `(src, target, ships, initial_by_id, ang_vel, comets, comet_ids) -> tuple or None` | `(angle, turns, pred_x, pred_y)` | **核心接口**：5 次迭代收敛拦截解 |
| `search_safe_intercept` | `(src, target, ships, initial_by_id, ang_vel, comets, comet_ids) -> tuple or None` | `(angle, turns, target_x, target_y)` | 时间窗口扫描，用于太阳阻挡场景 |

`aim_at` 算法：
1. 用目标当前位置估算首次到达
2. 预测舰队到达时目标的位置
3. 重新计算到新位置的到达
4. 重复直到收敛（最多 5 次迭代，容差 0.3 units + 1 turn）
5. 静态目标太阳阻挡时回退到 `search_safe_intercept`

### 3.1 types.py — 数据类型

```python
from src.world.types import GameState
```

```python
@dataclass
class GameState:
    step: int                          # 当前回合数
    player: int                        # 当前玩家 ID
    planets: list                      # list[Planet] (kaggle 命名元组)
    fleets: list                       # list[Fleet] (kaggle 命名元组)
    angular_velocity: float            # 轨道行星角速度
    initial_by_id: dict                # {planet_id: Planet at step 0}
    comets: list                       # 原始彗星组数据
    comet_ids: set                     # 彗星 planet IDs

    # ── 预计算分类列表 ──
    my_planets: list                   # owner == player
    enemy_planets: list                # owner not in (-1, player)
    neutral_planets: list              # owner == -1

    # ── 派生统计数据 ──
    remaining_steps: int               # episode_steps - step (最少为 1)
    episode_steps: int = 500           # 总回合数 (1v1=500, 4p=200)
    num_players: int = 2               # 活跃玩家数
    my_total_ships: int                # 我方行星+舰队总舰船
    enemy_total_ships: int             # 敌方行星+舰队总舰船
    my_total_production: int           # 我方行星总产量
    enemy_total_production: int        # 敌方行星总产量
```

**重要：** `enemy_*` 系列聚合所有非己方玩家（1v1 中即对手，4p 中为其他三人之和）。

### 4.1 builder.py — 决策矩阵组装（核心入口）

```python
from src.features import build_decision_matrix, DecisionRow
```

```python
@dataclass
class DecisionRow:
    source_id: int            # 源行星 ID
    candidate_id: int         # 目标行星 ID (-1 = no-op)
    self_feat: np.ndarray     # (21,)  float32
    cand_feat: np.ndarray     # (29,)  float32
    global_feat: np.ndarray   # (16,)  float32
    mask: bool                # True=有效, False=屏蔽
    action_info: dict         # {"angle": float, "ships": int, "eta": int, "available": int, ...}
```

| 函数 | 签名 | 返回值 | 说明 |
|------|------|--------|------|
| `build_decision_matrix` | `(state: GameState, candidate_count=20, sim_horizon=110) -> list[DecisionRow]` | 决策矩阵 | **Phase 2 核心输出** |

```python
# 使用示例
state = parse_observation(obs)
rows = build_decision_matrix(state, candidate_count=20)
# rows[0] 是行星 0 的 no-op 行 (candidate_id=-1)
# rows[1..K] 是行星 0 的 Top-K 候选目标
# rows 总长度 = Σ(1 + K_i) for each my_planet
```

**内部流程：**
1. `build_arrival_ledger(state.fleets, state.planets)` → ledger
2. `simulate_planet_timeline()` for each planet → timelines
3. `build_global_features(state, ledger)` → (16,)
4. 对每个己方行星：
   - `build_self_features(src, ...)` → (21,)
   - `_select_top_candidates()` → Top-K 候选
   - 对每个候选 `build_candidate_features(...)` → (29,)
   - 组装 DecisionRow

**no-op 保证：** 每个源行星的第一行是 `candidate_id=-1` 的 no-op 行，mask=True，cand_feat 全零。

| `ValueHead(hidden_dim)` | `(hidden_dim,)` | scalar | Linear(hidden_dim→1)，预测期望折现回报 |

### 5.3 model.py — 完整策略网络（Phase 5 ✅ 已完成）

**实际实现：**

```python
from src.policy.model import PolicyNetwork

policy = PolicyNetwork(hidden=256)

# forward() — 单源行星前向传播
out = policy.forward(self_feat, cand_feat, global_feat, mask)
# Returns: {"target_logits": (K,), "ship_alpha": scalar, "ship_beta": scalar, "value": scalar}

# forward_batch() — 批量前向传播（PPO 训练用）
out_b = policy.forward_batch(self_b, cand_b, global_b, mask_b)
# self_b: (B,21), cand_b: (B,K,29), global_b: (B,16), mask_b: (B,K)
# Returns: {"target_logits": (B,K), "ship_alpha": (B,), "ship_beta": (B,), "value": (B,)}

# act() — 训练/推理统一入口
out = policy.act(self_feat, cand_feat, global_feat, mask, deterministic=False)
# Returns: {"target_idx": int, "ship_ratio": float, "target_log_prob": float, ...}
```

**架构：**
```
SelfEncoder(21→256)            → self_hidden   (256d)
CandidateEncoder(29→256)       → cand_hidden   (K×256d)  [共享权重]
GlobalEncoder(16→256)          → global_hidden  (256d)

joint[i] = concat(self_hidden, global_hidden, cand_hidden[i])    # 768d
target_logit[i] = TargetHead(joint[i])

pooled_cand = masked_mean(cand_hidden, mask, dim=0)              # 256d
value_input = concat(self_hidden, global_hidden, pooled_cand)    # 768d
value = ValueHead(value_input)          # scalar
ship_alpha = ShipAlphaHead(value_input)  # scalar (>1)
ship_beta = ShipBetaHead(value_input)   # scalar (>1)
```

**参数量：** 218,116 (< 1M) | **初始化：** Orthogonal | **池化：** masked mean（排除无效候选）


|------|-----|------|

#### 函数

| 函数 | 签名 | 返回值 | 说明 |
|------|------|--------|------|
| `planet_remaining_life` | `(planet, state) -> int` | 剩余产出回合 | 彗星用 `comet_remaining_life()`，普通行星用 `remaining_steps` |
| `capture_present_value` | `(planet, state) -> float` | 未来总产出 | `remaining_life × production` (max=2500) |
| `normalize_pv` | `(raw_pv: float) -> float` | [0, 5] | `raw_pv / 500.0` |
| `state_potential` | `(state, player) -> float` | Φ 值 | 零和：我方 PV − 敌方 PV |
| `compute_step_reward` | `(prev_state, curr_state, player, game_ended=False, comet_warmup=0.0) -> float` | 奖励值 | **核心接口**：三步奖励 |
| `get_comet_warmup` | `(utilization_ema: float) -> float` | [0, 1.5] | 自适应 warmup |

#### 奖励结构（三层）

```
Layer 1: Δ舰船优势 = 3.0 × (curr_ship_ratio − prev_ship_ratio)
         对称编码代价信号（损兵自动惩罚）
Layer 2: ΔΦ_territory = Φ(s') − Φ(s)
         一行覆盖 6 种所有权变化，自动处理奖励/惩罚
Layer 3: 终局 ±10.0
+ comet_warmup: 刚占领的彗星额外加分（仅训练阶段）
```

**6 种所有权变化的 ΔΦ 信号：**

| 事件 | Δ(my_PV) | Δ(enemy_PV) | ΔΦ | 
|------|----------|-------------|-----|
| 我占中立 | +pv | 0 | **+pv** |
| 敌占中立 | 0 | +pv | **−pv** (自动惩罚) |
| 我抢敌星 | +pv | −pv | **+2pv** (天然双倍) |
| 敌抢我星 | −pv | +pv | **−2pv** (天然双倍惩罚) |
| 我失彗星 | −pv | 0 | **−pv** |
| 敌失彗星 | 0 | −pv | **+pv** (自动奖励) |

#### 彗星 warmup 自适应逻辑

```
utilization_ema = 0.99 × utilization_ema + 0.01 × attacked_from_comet_this_episode
warmup = 1.5 × (1 − min(1.0, utilization_ema / 0.20))
```

- EMA = 0.00 → warmup = 1.50 (全开引导)
- EMA = 0.10 → warmup = 0.75 (半开)
- EMA ≥ 0.20 → warmup = 0.00 (退场，值函数已自举)
- 行为退化时 EMA 回落 → warmup 自动恢复

## 附录 B: 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| Agent 不输出角度 | `(source, target, ratio)` | 物理引擎自动算 atan2 + ETA |
| 太阳绕行不实现 | 直飞 = 引擎行为 | 游戏引擎只允许直线飞行 |
| 距离→ETA 替代 | ETA 特征编码 | `fleet_speed(ships)` 非线性，网络不应重复学习 |
| 奖励 = ΔΦ | 纯 PBRS | 不引入偏差，数学可证不扭曲最优策略 |
| 对称势能 Φ | `my_PV − enemy_PV` | 敌人扩张自动惩罚，无盲区 |
| comet_warmup 自适应 | EMA 驱动 | 学会才退，退化自动恢复 |
| ship_ratio = Beta 分布 | 非 Gaussian+tanh | Beta 天然有界 [0,1] |
| CandidateEncoder 共享权重 | 排列不变性 | 候选间对称，降低过拟合 |
| no-op 作为普通候选 | `candidate_id=-1`, cand_feat=0 | 无需额外机制 |
| GAE 时间粒度 | per game step（非 per transition） | 同一 step 各行星共享 return，保持时序正确 |
| V_team 引导 + 独立 advantage | bootstrap=mean(V_i), A_i=R_step−V_i | 团队奖励共享但个体贡献分别评估 |
| forward_batch() | (B,K,D) flatten-reshape | GPU 并行 + 共享权重 CandidateEncoder |
| 单环境训练 | 1 env（非并行） | GAE 需要严格时序，多环境破坏 λ-return 定义 |
| Masked mean 池化 | 排除无效候选 | K 变长时的正确池化（Bug 修复：Issue #4 Phase 5） |
| Beta 熵可为负 | 不检查 >0，仅检查有限 | 微分熵不同于离散熵，Beta(α≫1,β≫1) 熵为负是正常行为 |

## 1. 玩家数量与主星分配

### 1v1 (2 players)
- 随机选一个 4 重对称的行星群组 (base × 4)
- Player 0 → base[0] (第一象限 Q1)
- Player 1 → base[3] (第四象限 Q4)
- 主星初始舰船 = 10

```python
# 引擎源码 L383-387
if num_agents == 2:
    obs0.planets[base][1] = 0      # Q1 → player 0
    obs0.planets[base][5] = 10
    obs0.planets[base + 3][1] = 1  # Q4 → player 1
    obs0.planets[base + 3][5] = 10
```

### 4p (4 players)
- 同样随机选一组，4 颗全部分配
- Player 0 → base[0] (Q1), Player 1 → base[1] (Q2), Player 2 → base[2] (Q3), Player 3 → base[3] (Q4)
- 4 重旋转对称保证公平：每组 4 颗行星互为 90° 旋转

```python
# 引擎源码 L388-391
elif num_agents == 4:
    for j in range(4):
        obs0.planets[base + j][1] = j
        obs0.planets[base + j][5] = 10
```

### 注意事项
- 其他未分配的行星 `owner = -1`（中立），初始舰船 5~99
- 所有玩家共享相同的棋盘和行星布局
- 唯一区别是各自的 `player` ID 和观察视角

## 2. 回合结构 (每步执行顺序)

所有玩家同时行动，引擎按固定顺序处理：

```
0. 舰队发射 (Fleet Launch)
   - 所有玩家的 action 同时处理
   - 舰船从源行星扣除，生成飞行中舰队
1. 生产 (Production)
   - 所有有主行星 (owner != -1) 生产舰船: ships += production
2. 行星移动 (Planet Movement)
   - 轨道行星绕太阳旋转
   - 彗星沿 path[] 前进一个索引
   - 静态行星不动
3. 舰队移动 + 碰撞检测 (Fleet Movement)
   - 连续 swept-pair 碰撞检测
   - 命中行星 → 加入该行星的战斗列表
   - 出界/撞太阳 → 销毁
4. 彗星过期移除
   - path_index >= len(paths[i]) 的彗星被移除
   - 移除发生在战斗结算之前
5. 战斗结算 (Combat Resolution)
   - 每颗行星独立结算
   - 多人战斗规则见 §3
6. 终局检查
   - step >= episodeSteps - 2 → 终局
   - 存活玩家 ≤ 1 → 终局
```

**关键：** 彗星过期 → 移除 → 战斗结算。占领者在最后回合无法撤回驻军。引擎源码 L409-429（过期移除在舰队发射前）和 L617-631（过期移除在行星移动后、战斗结算前）各执行一次。

## 3. 战斗结算 (多人战斗)

```python
# 引擎源码 L636-675
for pid, planet_fleets in combat_lists.items():
    # 1. 按 owner 聚合到达舰船
    player_ships = {}  # {owner: total_ships}
    for fleet in planet_fleets:
        player_ships[fleet.owner] += fleet.ships

    # 2. 按舰船数降序排列
    sorted_players = sorted(player_ships.items(), key=lambda x: x[1], reverse=True)

    # 3. 前两名对消
    top_player, top_ships = sorted_players[0]
    if len(sorted_players) > 1:
        second_ships = sorted_players[1][1]
        survivor_ships = top_ships - second_ships
        if top_ships == second_ships:
            survivor_ships = 0  # 平局 → 全部消灭
        survivor_owner = top_player if survivor_ships > 0 else -1
    else:
        survivor_owner = top_player
        survivor_ships = top_ships

    # 4. 幸存者 vs 驻军
    if survivor_ships > 0:
        if planet.owner == survivor_owner:
            planet.ships += survivor_ships       # 同主 → 合并
        else:
            planet.ships -= survivor_ships       # 不同 → 攻击
            if planet.ships < 0:
                planet.owner = survivor_owner    # 占领
                planet.ships = abs(planet.ships)
```

### 多人战斗示例

行星 X（owner=0, ships=50），同回合到达：
- Player 1: 30 ships
- Player 2: 20 ships
- Player 3: 10 ships

结果：
1. 排序: P1(30) > P2(20) > P3(10)
2. P1 vs P2: P1 剩 30-20=10
3. P3 的 10 艘不参战（已被 P1/P2 的对消"屏蔽"）
4. 幸存者 P1(10) vs 驻军(50): P1 剩 10 打不过 50 → 行星仍归 owner=0, ships=40

**重要特性：** 只有前两名参与对消。第三名及以后的舰队虽然到达了，但不参与对消也不被消灭——它们实际上被浪费了。这是引擎的真实行为。

## 4. 彗星机制 (1v1 与 4p 完全相同)

### 生成时机
```python
COMET_SPAWN_STEPS = [50, 150, 250, 350, 450]  # 硬编码，不随玩家数缩放
```

| 模式 | episodeSteps | 触发的彗星波次 | 触发回合 |
|------|-------------|---------------|---------|
| 1v1 | 500 | 5 波 | 50, 150, 250, 350, 450 |
| 4p | 200 | 2 波 | 50, 150 |

250/350/450 三波在 4p 中超出了一局游戏的长度，永不触发。

### 彗星属性 (所有模式相同)
- `radius = 1.0`
- `production = 1` (固定值)
- `speed = 4.0` (沿路径移动)
- 生成: 4 重旋转对称，每波 4 颗
- 在轨寿命: 5~40 回合 (由 path 长度决定)
- 初始舰船: `min(rng(1,99), rng(1,99), rng(1,99), rng(1,99))` (4 次随机取最小)

### 彗星生命周期
1. `step+1 in COMET_SPAWN_STEPS` → 生成 4 颗 off-board 彗星行星
2. 下一回合：`path_index++`，放置到 path[0]
3. 每回合沿 path 前进，直到 `path_index >= len(path)` → 过期
4. 过期移除发生在战斗结算之前，占领者无法撤回驻军

## 5. 终局判定

```python
# 引擎源码 L689-697
alive_players = set()
for p in obs0.planets:
    if p.owner != -1:
        alive_players.add(p.owner)
for f in obs0.fleets:
    alive_players.add(f.owner)

if len(alive_players) <= 1:
    terminated = True
```

- 条件 1: `step >= episodeSteps - 2` → 终局
- 条件 2: 存活玩家数 ≤ 1 → 终局 (某玩家舰队+行星全灭)
- 分数 = 行星驻军 + 飞行中舰队 (按 owner 求和)
- 最高分者 reward=1，其余 reward=-1
- 平局（多个最高分）→ 都 reward=1

## 6. 舰船速度

```python
speed = 1.0 + (max_speed - 1.0) * (log(ships) / log(1000)) ^ 1.5
# max_speed = configuration.shipSpeed (默认 6.0)
```

- 1 艘舰船: speed ≈ 1.0
- 大舰队: speed → 6.0
- 与玩家数无关，公式完全一致

### 仍存在的 1v1 简化
| 模块 | 简化内容 | 4p 影响 |
|------|---------|---------|
| `GameState.enemy_planets` | 聚合所有对手行星 | 无法区分对手 |

### 直接支持 4p 的模块
- `combat.py` — 多人战斗结算正确 (前两名对消)
- `fleet_tracker.py` — 无玩家数假设
- `physics.py`, `prediction.py`, `interception.py` — 纯物理计算
- `action_head.py`, `value_head.py` — 纯网络组件
- `state_potential()` — 正确聚合所有敌对 owner
- 引擎本身 — 原生支持 2p/4p

## 8. 4p 扩展路线图

当需要支持 4 人乱斗时，需要做以下改动：

### 必须改动
1. **特征工程** — 区分对手 1/2/3（或按威胁度排序），新增 per-opponent 特征
2. **Φ 函数** — 替换为适合 4p FFA 的形式，如：
   - `Φ = my_PV − max(other_players'_PV)` (最强者威胁)
   - `Φ = my_PV − average(other_players'_PV)` (平均优势)
3. **终局奖励** — 4p 终局是排名制（1st/2nd/3rd/4th），不是胜负制
4. **训练** — 需要 self-play 或多对手池

### 可选改动
5. 候选选择策略 — 优先攻击领先者而非最近的敌人
6. 外交/联盟特征 — 暂不推荐，过于复杂

# Orbit Wars（轨道战争）

在围绕太阳旋转的连续 2D 空间中征服行星。这是一款支持 2 人或 4 人参与的实时策略游戏。

## 概述

玩家从一颗母星开始，通过派遣舰队占领中立和敌方行星来争夺地图控制权。棋盘为 100x100 的连续空间，中心有一颗太阳。行星绕太阳运行，彗星沿椭圆轨道飞过，舰队沿直线航行。游戏持续 500 回合。结束时，总舰船数（行星驻军 + 舰队中的舰船）最多的玩家获胜。

## 棋盘布局

- **棋盘**：100x100 连续空间，原点在左上角。
- **太阳**：位于中心 (50, 50)，半径 10。穿越太阳的舰队会被摧毁。
- **对称性**：所有行星和彗星围绕中心呈 4 重镜像对称放置：(x, y)、(100-x, y)、(x, 100-y)、(100-x, 100-y)。这确保了无论起始位置如何都公平。

## 行星

每颗行星表示为 `[id, owner, x, y, radius, ships, production]`。

- **owner**：玩家 ID（0-3），-1 表示中立。
- **radius**：由产量决定：`1 + ln(production)`。产量越高的行星物理尺寸越大。
- **production**：1 到 5 的整数。每回合，有主行星会生成这么多舰船。
- **ships**：当前驻军数量。初始值在 5 到 99 之间（偏向较低值）。

### 行星类型

- **轨道行星**：满足 `orbital_radius + planet_radius < 50` 的行星以恒定角速度（0.025-0.05 弧度/回合，每局随机）绕太阳旋转。使用观测中的 `initial_planets` 和 `angular_velocity` 来预测其位置。
- **静态行星**：距离中心较远的行星不旋转。

地图包含 20-40 颗行星（5-10 个对称组，每组 4 颗）。至少 3 组保证为静态行星，至少 1 组保证为轨道行星。

### 母星

随机选择一个对称组作为起始行星。在 2 人游戏中，玩家在对角线相对的行星上开始（Q1 和 Q4）。在 4 人游戏中，每位玩家获得该组中的一颗行星。母星初始拥有 10 艘舰船。

## 舰队

每支舰队表示为 `[id, owner, x, y, angle, from_planet_id, ships]`。

- **angle**：航行方向（弧度制）。
- **ships**：舰队中的舰船数量（航行中不变）。

### 舰队速度

舰队速度随规模呈对数曲线变化：

```
speed = 1.0 + (maxSpeed - 1.0) * (log(ships) / log(1000)) ^ 1.5
```

- 1 艘舰船以 1.0 单位/回合移动。
- 较大舰队移动更快，趋近于最大速度（默认 6.0）。
- 约 500 艘舰船的舰队移动速度约 5，约 1000 艘舰船达到最大速度。

### 舰队移动

舰队每回合沿直线以计算出的速度航行。舰队在以下情况下被移除：

- 出界（离开 100x100 游戏区域）。
- 穿越太阳（路径线段进入太阳半径范围）。
- 与任意行星碰撞（路径线段进入行星半径范围）。这触发战斗。

碰撞检测是连续的——检查从旧位置到新位置的整条路径线段，而非仅检查终点。

### 舰队发射

每回合，您的智能体返回一个行动列表：`[from_planet_id, direction_angle, num_ships]`。

- 只能从您拥有的行星发射。
- 不能发射超过行星当前拥有的舰船数。
- 舰队在行星半径刚好外沿给定方向生成。
- 您可以在单回合内从同一行星或不同行星发射多支舰队。

## 彗星

彗星是临时的太阳系外天体，沿绕太阳的高度椭圆轨道穿越棋盘。它们在第 50、150、250、350 和 450 回合以每组 4 颗（每个象限一颗）生成。

- **半径**：1.0（固定）。
- **产量**：被占领时每回合 1 艘舰船。
- **初始舰船**：随机，偏向较低值（至少从 1-99 中掷 4 次）。同组所有 4 颗彗星共享相同的初始舰船数。
- **速度**：可通过 `cometSpeed` 配置（默认 4.0 单位/回合）。
- **识别**：在观测中检查 `comet_planet_ids` 以查看哪些行星 ID 是彗星。彗星也会出现在 `planets` 数组中，遵循所有正常的行星规则（占领、生产、舰队发射、战斗）。

彗星离开棋盘时，其上的驻军舰船也会一并移除。彗星在每回合舰队发射前被移除，因此无法从即将离开的彗星发射舰队。

观测中的 `comets` 字段包含彗星组数据，包括 `paths`（每颗彗星的完整轨迹）和 `path_index`（当前沿路径的位置），可用于预测彗星未来位置。

## 回合顺序

每回合按以下顺序执行：

1. **彗星过期**：移除已离开棋盘的彗星。
2. **彗星生成**：在指定回合生成新的彗星组。
3. **舰队发射**：处理所有玩家行动，创建新舰队。
4. **生产**：所有有主行星（包括彗星）生成舰船。
5. **舰队移动**：所有舰队沿其方向移动。检查出界、太阳碰撞和行星碰撞。撞击行星的舰队排入战斗队列。
6. **行星旋转与彗星移动**：轨道行星旋转，彗星沿其路径前进。任何被移动行星/彗星扫到的舰队被卷入与其战斗。
7. **战斗结算**：结算所有排队的行星战斗。

## 战斗

当一支或多支舰队与行星碰撞（飞入或被移动行星扫到），战斗结算如下：

1. 所有到达的舰队按拥有者分组。同一拥有者的舰船求和。
2. 最大的攻击方与第二大的攻击方对战。差值舰船数幸存。
3. 如果有幸存攻击方：
   - 若攻击方与行星拥有者相同，幸存舰船加入驻军。
   - 若攻击方不同，幸存舰船与驻军对战。若攻击方超过驻军，行星易主，驻军变为溢出的舰船数。
4. 若两方攻击打平，所有攻击舰船被摧毁（无幸存者）。

## 计分与终止

游戏在以下情况下结束：

- **达到回合上限**：500 回合。
- **淘汰**：只剩一位玩家（或零位）拥有行星或舰队。

最终分数 = 拥有行星上的总舰船 + 拥有舰队中的总舰船。最高分获胜。

## 观测参考

| 字段 | 类型 | 描述 |
|------|------|------|
| `planets` | `[[id, owner, x, y, radius, ships, production], ...]` | 所有行星（含彗星） |
| `fleets` | `[[id, owner, x, y, angle, from_planet_id, ships], ...]` | 所有活动舰队 |
| `player` | `int` | 您的玩家 ID（0-3） |
| `angular_velocity` | `float` | 行星旋转速度（弧度/回合） |
| `initial_planets` | `[[id, owner, x, y, radius, ships, production], ...]` | 游戏开始时的行星位置 |
| `comets` | `[{planet_ids, paths, path_index}, ...]` | 活动彗星组数据 |
| `comet_planet_ids` | `[int, ...]` | 作为彗星的行星 ID |
| `remainingOverageTime` | `float` | 剩余超时预算（秒） |

## 行动格式

返回行动列表：

```python
[[from_planet_id, direction_angle, num_ships], ...]
```

- `from_planet_id`：您拥有的行星的 ID。
- `direction_angle`：弧度制角度（0 = 向右，pi/2 = 向下）。
- `num_ships`：要发送的舰船整数数量。

返回空列表 `[]` 表示不采取行动。

## 智能体便利工具

该模块导出命名元组以便于字段访问：

```python
from kaggle_environments.envs.orbit_wars.orbit_wars import Planet, Fleet, CENTER, ROTATION_RADIUS_LIMIT

def agent(obs):
    planets = [Planet(*p) for p in obs.get("planets", [])]
    fleets = [Fleet(*f) for f in obs.get("fleets", [])]
    player = obs.get("player", 0)

    for p in planets:
        print(p.id, p.owner, p.x, p.y, p.radius, p.ships, p.production)

    return []  # list of [from_planet_id, angle, num_ships]
```
